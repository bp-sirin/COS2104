How CPU Registers Work

รีจิสเตอร์ (Register) คืออะไร?

รีจิสเตอร์ คือหน่วยความจำขนาดเล็กมากที่สร้างขึ้นอยู่ภายในตัวประมวลผลกลาง (CPU) โดยตรง ถือเป็นหน่วยความจำที่ทำงานด้วย **ความเร็วสูงที่สุด** ในระบบคอมพิวเตอร์ ทำหน้าที่เสมือนกับ "โต๊ะทำงานของช่าง" เพื่อประมวลผลข้อมูลชั่วคราว

💡 เปรียบเทียบความแตกต่างการทำงาน

Register (รีจิสเตอร์): เครื่องมือช่างที่ถือไว้ในมือ หยิบใช้งานได้ทันทีใน 1 รอบสัญญาณนาฬิกา
Cache Memory (แคช L1-L3): ชั้นวางเครื่องมือข้างหลังโต๊ะทำงาน หยิบใช้งานได้เร็วปานกลาง
RAM (หน่วยความจำหลัก): โกดังเก็บของที่อยู่นอกตัวอาคาร มีขนาดใหญ่มาก แต่การหยิบของต้องใช้เวลาเดินเดินทาง (Latency สูง)

ทำไมรีจิสเตอร์ถึงเร็วที่สุด?

รีจิสเตอร์ถูกสร้างขึ้นโดยใช้การต่อวงจรทรานซิสเตอร์แบบ Flip-Flop (ประมาณ 6 ตัวต่อการเก็บข้อมูล 1 บิต) ซึ่งมีความรวดเร็วทางฟิสิกส์สูงมาก ทำงานได้ตรงตามสัญญาณนาฬิกาโดยไม่ต้องประจุไฟซ้ำ (Refresh) เหมือนหน่วยความจำ DRAM ที่ใช้ใน RAM ทั่วไป ยิ่งกว่านั้นมันยังตั้งอยู่ใกล้กับ ALU (Arithmetic Logic Unit) มากที่สุด ทำให้ข้อมูลเดินทางผ่านกระแสไฟฟ้าเกือบจะทันที

ข้อจำกัดที่เราไม่มี Register เป็นกิกะไบต์ (GB)

เนื่องจากวงจร Flip-Flop กินพื้นที่ชิปขนาดใหญ่มาก หากผลิตรีจิสเตอร์จำนวนมากจะส่งผลให้ตัวชิปมีขนาดกว้างเกินไปและเกิดความร้อนสะสมที่สูงมากจนพังเสียหาย นอกจากนี้ชุดคำสั่ง (Instruction Set) จะต้องใช้บิตข้อมูลแอดเดรสที่ยาวมากเพื่อระบุตำแหน่งรีจิสเตอร์ ทำให้การประมวลผลภาพรวมช้าลง

🔄 Register Spilling คืออะไร?

เมื่อเขียนโปรแกรม ตัวแปลภาษา (Compiler) จะคอยจัดสรรว่าตัวแปรใดสำคัญสุดให้ขึ้นไปรันบนรีจิสเตอร์ หากมีตัวแปรเกินขีดจำกัด คอมไพเลอร์จะเขียนคำสั่งเพื่อคัดลอกข้อมูลเก่าลงไปเก็บไว้ใน RAM ชั่วคราว (เรียกขั้นตอนนี้ว่า Register Spilling หรือ Spill to Stack) เพื่อเว้นพื้นที่รีจิสเตอร์ให้ข้อมูลชุดใหม่

สถาปัตยกรรม x86-64 Registers

สถาปัตยกรรม x86-64 เป็นสถาปัตยกรรมแบบ CISC (Complex Instruction Set Computer) มีโครงสร้างรีจิสเตอร์ที่พัฒนาต่อเนื่องมาหลายสิบปีตั้งแต่ยุค 8-bit จึงมีกลไกย่อยที่สามารถเข้าถึงบิตล่างหรือบนได้

General-Purpose Registers (GPRs)

มีรีจิสเตอร์ใช้งานทั่วไป 16 ตัวที่สามารถระบุตำแหน่งแบบ 64 บิต (`R` prefix), 32 บิต (`E` prefix), 16 บิต, หรือ 8 บิตได้

⚠️ ข้อควรจำ: Segment Registers

ในอดีต `CS`, `DS`, `SS`, `ES`, `FS`, `GS` เคยทำหน้าที่แบ่งแอดเดรสหน่วยความจำ แต่ในปัจจุบันระบบปฏิบัติการสมัยใหม่ใช้โมเดลหน่วยความจำแบนราบ (Flat Memory Model) เซกเมนต์รีจิสเตอร์เหล่านี้จึงใช้ชี้ไปยังข้อมูลการทำงานเฉพาะตัวของระบบ เช่น Thread-Local Storage (TLS) เท่านั้น

ARM64 (Apple Silicon Mx) Registers

ชิป Apple Silicon (M1 - M5) ทำงานบนสถาปัตยกรรม ARM64 (AArch64) ซึ่งเป็นแบบ RISC (Reduced Instruction Set Computer) มีการจัดสรรรีจิสเตอร์ใช้งานทั่วไปอย่างเป็นระบบถึง 31 ตัว (X0 - X30) ซึ่งเรียบง่ายและเป็นระบบมากกว่า x86

รีจิสเตอร์ใช้งานทั่วไปและบทบาททางกฎระเบียบ (ARM ABI)

🚀 ส่วนเสริมสำหรับ AI: SVE & SME ใน Mx รุ่นใหม่

ตั้งแต่ชิป Apple M3, M4, M5 ขึ้นไป จะนำสถาปัตยกรรม ARMv9-A มาใช้ ซึ่งรวมเอาชุดรีจิสเตอร์เวกเตอร์ปรับขนาดได้ Z0 - Z31 (SVE) และรีจิสเตอร์สะสมเมทริกซ์ 2 มิติ ZA (SME) เข้ามาเร่งความเร็วการประมวลผลปัญญาประดิษฐ์ (ML/AI) ช่วยลดเวลาบวกและคูณแมทริกซ์อย่างมหาศาล

CPU Datapath & Control Registers

ในระบบคอมพิวเตอร์ นอกเหนือจากรีจิสเตอร์ใช้งานทั่วไป (GPRs) แล้ว ยังมีรีจิสเตอร์ระบบภายในที่ช่วยประสานงานควบคุมทิศทางของข้อมูลระหว่าง CPU Core, ALU (หน่วยประมวลผลคำนวณ) และ System RAM

ทำความเข้าใจเรื่องเส้นทางข้อมูล (Datapath)

ALU (Arithmetic Logic Unit): เปรียบเสมือน "เครื่องคิดเลขฟิสิกส์" ที่ประมวลผลบวก ลบ และตรรกศาสตร์ แต่ตัวมันไม่มีสมองในตัวที่จะบันทึกค่าไว้ได้ สัญญาณผลลัพธ์ที่ออกจะวิ่งออกไปที่อื่นทันที ดังนั้น CPU จึงต้องมีหน่วยควบคุมและโครงข่ายรีจิสเตอร์พักข้อมูลคอยจัดส่งพิกัดและพัสดุข้อมูลให้ถูกที่

รู้จักผู้ประสานงานหลักทั้ง 4 ใน CPU Core

PC (Program Counter) 16 / 32 / 64-bit
ผู้ถือแผนที่นำทาง 🗺️
ทำหน้าที่เก็บแอดเดรสตำแหน่งหน่วยความจำของคำสั่งถัดไปที่โปรแกรมกำลังจะเรียกเข้าทำงาน โดยค่าจะขยับขึ้นอัตโนมัติเมื่อดึงข้อมูลไปแล้ว หรือเปลี่ยนแอดเดรสเมื่อพบคำสั่ง Jump/Branch
MAR (Memory Address Register) 16 / 32 / 64-bit
คนส่งพัสดุฝั่งที่อยู่ ✉️
เก็บตำแหน่งของหน่วยความจำแรม (RAM Address) ที่ CPU กำลังต้องการเข้าไปเขียนข้อมูล หรือเปิดอ่านข้อมูล เชื่อมต่อตรงกับบัสแอดเดรส (Address Bus)
MBR / MDR 16 / 32 / 64-bit
ท่าพักตู้คอนเทนเนอร์ 📦
ทำหน้าที่เก็บตัวข้อมูลจริง ๆ (Data) ที่เพิ่งรับอ่านมาจากแรม หรือข้อมูลที่เตรียมส่งบันทึกลงแรมเพื่อรอการสลับสับเปลี่ยน เชื่อมต่อตรงกับบัสข้อมูล (Data Bus)
IR (Instruction Register) 16 / 32 / 64-bit
โต๊ะแกะกล่องคำสั่ง 📋
เก็บรหัสคำสั่ง (Instruction Code) ที่ CPU ไปดึงมาจากแรมเพื่อส่งให้หน่วยควบคุม (Control Unit) แปลความหมายและสั่งทำงานต่อไป
🔄 ลำดับขั้นการ Fetch-Decode-Execute ในหนึ่งรอบคำสั่ง

1. Fetch: ค่าแอดเดรสใน PC จะถูกก๊อปปี้ไปที่ MAR -> แรมนำรหัสคำสั่งกลับมาไว้ที่ MBR -> ก๊อปปี้รหัสคำสั่งไปที่ IR และขยับค่าใน PC เพิ่มขึ้น
2. Decode: หน่วยควบคุมมาเปิดคำสั่งใน IR เพื่อตีความและเปิดปิดวาล์วส่งกระแสไฟสัญญาณให้ทำงานถูกต้อง
3. Execute: โหลดตัวแปรผ่าน MAR เข้ามาที่ MBR -> ส่งเข้าประมวลผลที่ ALU -> นำผลลัพธ์เขียนกลับลงรีจิสเตอร์ที่กำหนด

ตารางเปรียบเทียบ & เครื่องมือวิเคราะห์เทียบเคียง

ตารางเปรียบเทียบสถาปัตยกรรมหลักของเครื่องควบคุมรีจิสเตอร์ระหว่าง x86-64 และ ARM64 (Apple Silicon Mx)

หัวข้อ x86-64 (Intel / AMD) ARM64 (Apple Silicon Mx)
จำนวนรีจิสเตอร์ใช้งานทั่วไป (GPRs) 16 ตัว (RAX ถึง RSP, R8 ถึง R15) 31 ตัว (X0 ถึง X30)
รีจิสเตอร์ค่าศูนย์ทางฮาร์ดแวร์ ไม่มี (ต้องใช้ XOR เคลียร์แอดเดรส) มี (รีจิสเตอร์ XZR และ WZR มีค่าเป็น 0 เสมอ)
การเรียกใช้งานฟังก์ชัน (Calling Convention) ใช้บิตในแรม Stack บันทึกแอดเดรสกลับทันทีผ่าน `CALL` ใช้บิตรีจิสเตอร์ `X30` (Link Register) บันทึกแอดเดรสกลับผ่าน `BL`
โครงสร้างเวกเตอร์คณิตศาสตร์ (SIMD) XMM (128-bit), YMM (256-bit), ZMM (512-bit) V0-V31 (128-bit NEON), Z0-Z31 (SVE), ZA (SME)
สถาปัตยกรรมหน่วยความจำ CISC (ทำคำสั่งบวกกับแรมได้โดยตรง) Load-Store (ต้องดึงข้อมูลใส่รีจิสเตอร์ก่อนประมวลผล)

⚡ ทำไม RISC ถึงไม่มีคำสั่งนี้?
